# 地面静止时位置漂移分析

## 一、问题描述

飞机静止放在地面，过一段时间地面站显示位置漂移。这个位置是谁提供的？漂移是 GPS 导致的还是 IMU 导致的？

---

## 二、核心结论

**地面显示的位置 = EKF 融合后的估计位置，不是原始 GPS 位置。**

漂移可能来自 GPS 本身，也可能来自 EKF 估计，要分情况：

| 情况 | 漂移原因 |
|------|---------|
| 正常静止 | **GPS 本身漂移**（几米范围随机游走） |
| 被认为在动 | **IMU 积分漂移**（零速约束未触发） |
| GPS 丢失 | **IMU 积分漂移**（必然发生） |
| GPS glitch | **GPS 跳变**（EKF 可能拒绝或跟随） |

---

## 三、GPS 本身漂移（最常见）

### 3.1 ArduPilot 的 GPS 漂移检测

文件：`libraries/AP_NavEKF3/AP_NavEKF3_VehicleStatus.cpp:48-76`

```cpp
// Check for significant change in GPS position if disarmed which indicates bad GPS
// This check can only be used when the vehicle is stationary

// Sum distance moved
gpsDriftNE += gpsloc_prev.get_distance(gpsloc);
gpsloc_prev = gpsloc;

// Decay distance moved exponentially to zero
gpsDriftNE *= (1.0f - deltaTime/posFiltTimeConst);

// Fail if more than 3 metres drift after filtering whilst on-ground
bool gpsDriftFail = (gpsDriftNE > 3.0f*checkScaler) && onGround;
```

**说明**：ArduPilot 明确知道 GPS 在地面静止时也会漂移，阈值是 **3 米**。

### 3.2 GPS 漂移原因

- 多路径效应（信号反射）
- 卫星几何分布变化（DOP 变化）
- 大气层延迟变化
- 普通 GPS 精度 2-3 米，RTK 才能到厘米级

## 四、EKF 的零速约束机制

### 4.1 零速约束的作用

文件：`libraries/AP_NavEKF3/AP_NavEKF3_PosVelFusion.cpp:699-718`

```cpp
// When stationary on ground or armed before takeoff, fuse zero velocity
// to constrain gyro bias and Z-axis accel bias learning.

const bool onGroundNotFlying = onGroundNotMoving || dal.get_takeoff_expected();
if (onGroundNotFlying && tiltAlignComplete) {
    fuseVelData = true;
    fusingStationaryZeroVel = true;
    velPosObs[0] = 0.0f;  // 强制速度为 0
    velPosObs[1] = 0.0f;
    velPosObs[2] = 0.0f;
}
```

**注意**：零速约束融合的是 **velocity = 0**，不是 **position = 固定值**。

### 4.2 零速约束为什么不防止位置漂移？

```
速度 = 0 意味着：位置不变化
但位置本身可以是任意值
```

- 如果 GPS 说位置在 (2, 1)，EKF 融合 GPS 位置 → 位置变成 (2, 1)
- 零速度约束 → 速度变成 0
- 结果：位置停在 (2, 1)，不再动
- **但如果 GPS 下一秒说位置在 (3, 2)，EKF 会跟着漂**

### 4.3 无速度辅助时的零速注入

文件：`libraries/AP_NavEKF3/AP_NavEKF3_PosVelFusion.cpp:722-754`

```cpp
// When in AID_RELATIVE or AID_ABSOLUTE mode but stationary on ground
// without velocity aiding, fuse synthetic zero velocity to constrain
// gyro bias and Z-axis accel bias learning.

if (!haveRecentGpsVel && !haveRecentFlowVel && !haveRecentBodyVel) {
    // No velocity aiding available while stationary - fuse synthetic zero velocity
    fuseVelData = true;
    fusingStationaryZeroVel = true;
    velPosObs[0] = 0.0f;
    velPosObs[1] = 0.0f;
    velPosObs[2] = 0.0f;
}
```

## 五、onGroundNotMoving 判定逻辑

### 5.1 判定条件

文件：`libraries/AP_NavEKF3/AP_NavEKF3_Measurements.cpp:1394-1454`

飞控通过 **4 个指标** 判断飞机是否在地面静止：

| 指标 | 阈值 | 含义 |
|------|------|------|
| `gyro_length_ratio` | 3.0°/s | 陀螺仪角速度大小 |
| `accel_length_ratio` | 1.0 m/s² | 加速度计偏离重力的大小 |
| `gyro_diff_ratio` | 0.2 rad/s² | 陀螺仪角加速度（变化率） |
| `accel_diff_ratio` | 5.0 m/s³ | 加速度计变化率 |

### 5.2 判定公式

```cpp
// 参数：EK3_OGNM_TEST_SF（默认 2.0）

// 当前认为在动 -> 判断是否需要改为"静止"
if (onGroundNotMoving) {
    // 任一指标超过阈值 × 缩放因子，认为在动
    if (gyro_length_ratio > 2.0 ||
        accel_length_ratio > 2.0 ||
        gyro_diff_ratio > 2.0 ||
        accel_diff_ratio > 2.0)
    {
        onGroundNotMoving = false;
    }
} else {
    // 当前认为在动 -> 判断是否可以改为"静止"
    // 所有指标必须同时低于阈值 × 缩放因子 × 0.7（滞回）
    if (gyro_length_ratio < 1.4 &&
        accel_length_ratio < 1.4 &&
        gyro_diff_ratio < 1.4 &&
        accel_diff_ratio < 1.4)
    {
        onGroundNotMoving = true;
    }
}
```

### 5.3 什么情况下会误判为"在动"？

| 误判原因 | 具体表现 | 解决 |
|---------|---------|------|
| **风吹/人为触碰** | 陀螺仪角速度超标（> 6°/s） | 把飞机放室内/避风处 |
| **IMU 零偏未收敛** | 刚上电时一直显示在动 | 等待几分钟让 EKF 收敛 |
| **斜坡/倾斜放置** | 加速度计读数偏离重力（> 2.0 m/s²） | 放水平地面 |
| **振动过大** | 变化率指标超标 | 检查减震、螺旋桨平衡 |
| **参数太敏感** | 轻微晃动就触发 | 调大 `EK3_OGNM_TEST_SF` |

## 六、IMU 导致漂移的情况

### 6.1 零速约束失效时

1. **飞机被认为在移动**（`onGroundNotMoving = false`）
   - 有人在推飞机
   - 在船上
   - 风大导致晃动

2. **没有高度数据源**（`fuseHgtData = false`）
   - 气压计故障
   - 高度融合超时

3. **GPS 完全丢失**且没有其他 aiding
   - 进入纯惯性导航
   - IMU 积分必然漂移

### 6.2 为什么正常静止时 IMU 不会导致漂移？

因为：
- 零速约束强制速度 = 0
- EKF 会估计并扣除 gyro_bias / accel_bias
- 位置跟随 GPS，不依赖 IMU 积分

---

## 七、如何区分漂移原因？

| 判断方法 | GPS 漂移 | EKF/IMU 漂移 |
|---------|---------|-------------|
| **看原始 GPS 坐标** | 原始 GPS 也在变 | 原始 GPS 稳定，EKF 输出在变 |
| **看 EKF 状态** | `gpsGlitching = false` | `gpsGlitching = true` 或 innovations 很大 |
| **拔掉 GPS** | 位置不再更新 | EKF 进入纯惯性导航，快速漂移 |
| **看速度** | 速度接近 0（有零速约束） | 速度不为 0（IMU 积分导致） |
| **看 innovation** | `posInnov` 小 | `posInnov` 大 |
| **看日志 XKFM** | `ongroundnotmoving = true` | `ongroundnotmoving = false` |

## 八、GPS Glitch 检测

文件：`libraries/AP_NavEKF3/AP_NavEKF3_PosVelFusion.cpp:932-1070`

```cpp
// GPS 融合时的 glitch 检测
if (frontend->_gpsGlitchRadiusMax > 0) {
    // 如果 GPS 位置和惯性估计差太远，认为是 glitch
    if (gpsInnovSq > sq(frontend->_gpsGlitchRadiusMax)) {
        // 拒绝这次 GPS 数据，不融合
        gpsDataDelayed.bad_fix = true;
    }
}
```

**参数**：`EK3_GLITCH_RAD`（默认 0 = 禁用）

如果 GPS 突然跳变，EKF 会检测到这个 glitch 并拒绝融合。

## 九、总结

| 问题 | 答案 |
|------|------|
| **地面显示的位置是谁的？** | EKF 融合后的估计位置，不是原始 GPS |
| **漂移主要是谁的锅？** | **通常是 GPS 本身漂移**（几米范围随机游走） |
| **IMU 会导致漂移吗？** | 会，但 EKF 有**零速约束**防止地面漂移 |
| **什么情况下 IMU 会导致漂移？** | `onGroundNotMoving = false` 时（被认为在动） |
| **怎么判断？** | 对比原始 GPS 和 EKF 输出，看 innovations，看日志 |
| **怎么解决？** | 1. 用 RTK GPS<br>2. 检查 GPS 天线位置<br>3. 调整 `EK3_GLITCH_RAD`<br>4. 等待 GPS 收敛（冷启动需要几分钟）<br>5. 确保飞机放水平、避风、无振动 |

**一句话**：地面静止时看到的位置漂移，**绝大多数情况是 GPS 本身在几米范围内随机游走**。正常静止时零速约束会防止 IMU 导致的速度/位置漂移。只有当飞控误判飞机"在动"（`onGroundNotMoving = false`）时，IMU 积分才会导致漂移。
