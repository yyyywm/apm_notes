# 02 — 飞行模式系统

> 文件: `ArduPlane/mode.h` + `mode.cpp` + 25 个 `mode_*.cpp`
> 核心: Mode 基类定义了飞行模式的统一接口，26 种模式各自实现

---

## Mode 基类

### 生命周期方法

| 方法 | 调用者 | 触发时机 | 用途 |
|------|--------|---------|------|
| `enter()` | `set_mode()` | 模式切换时 | 调用 `_enter()` 初始化 + 重置公共状态 |
| `_enter()` | `enter()` | 模式切换时 | 子类重写，初始化该模式特有状态 |
| `exit()` | `set_mode()` | 模式切换时 | 调用 `_exit()` 清理 |
| `_exit()` | `exit()` | 模式切换时 | 子类重写，保存/清理 |

### 运行方法

| 方法 | 调用者 | 频率 | 用途 |
|------|--------|------|------|
| `update()` | `update_control_mode()` | FAST_TASK (每圈) | **导航层**: 计算导航需求 (roll/pitch/throttle 目标) |
| `run()` | `stabilize()` | FAST_TASK (每圈) | **稳定层**: PID 控制 (stabilize_roll/pitch/yaw) |
| `navigate()` | `navigate()` | 10Hz | 航点追踪 (仅 auto/rtl/guided 等有自动导航的模式) |

### 能力查询 (virtual, 子类可 override)

- `is_vtol_mode()` — 是否为 VTOL 模式
- `is_guided_mode()` — 是否为 Guided 类模式
- `does_auto_navigation()` — 是否自动导航
- `does_auto_throttle()` — 是否自动油门
- `allows_throttle_nudging()` — 是否允许油门微调
- `allows_terrain_disable()` — 是否允许禁用地形跟随
- `is_landing()` — 是否处于降落状态
- `is_taking_off()` — 是否处于起飞状态

### run() 默认实现

```
1. STICK_MIXING 混控 (根据 STICK_MIXING 参数选择 NONE/FBW/VTOL_YAW/FBW_NO_PITCH)
2. stabilize_roll()  → 滚转 PID → 副翼
3. stabilize_pitch() → 俯仰 PID → 升降
4. stabilize_yaw()   → 偏航 PID (协调转弯/航向保持/地面转向)
```

## set_mode() 切换流程

→ 文件: `ArduPlane/system.cpp`

```
set_mode(Mode::Number new_mode, ModeReason reason)
  │
  ├── 1. 验证 new_mode 合法性
  │   ├── VTOL 模式需要 Q_ENABLE ≥ 1
  │   └── 无 GPS 时拒绝需要 GPS 的模式
  │
  ├── 2. 调用 new_mode.enter()
  │   └── 内部: 重置公共状态 → 调用 _enter()
  │
  ├── 3. 如果 enter 成功:
  │   ├── old_mode.exit()  → 调用 _exit()
  │   ├── control_mode = &new_mode
  │   ├── previous_mode = old_mode
  │   └── 日志记录 + GCS 通知
  │
  └── 4. 如果 enter 失败 → 保持原模式
```
